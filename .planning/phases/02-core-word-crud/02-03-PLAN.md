## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| WordDetailDrawer edit form → PATCH API | User can modify any field of their word |
| Delete confirm flow → DELETE API | Irreversible action requires explicit double-click confirmation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Elevation of Privilege | useUpdateWord (PATCH) | mitigate | Backend ownership check in getWordForUser(userId, wordId) returns 403 before update |
| T-02-11 | Elevation of Privilege | useDeleteWord (DELETE) | mitigate | Same backend ownership check before delete |
| T-02-12 | Tampering | Edit form difficulty field | accept | Values constrained to enum options in select; invalid values rejected by Prisma at DB level |
| T-02-13 | Repudiation | Delete without confirmation | mitigate | Two-step confirm-delete mode required (D-16); no accidental deletion |
</threat_model>
