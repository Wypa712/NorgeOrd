<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → /api/words | Untrusted input: headword, translation, gender, wordClass, notes, difficulty, personalNote |
| JWT token → requireAuth | Untrusted identity claim decoded from Bearer header |
| req.user.userId → Prisma query | userId used directly in WHERE clause — must not be overridable by body |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | requireAuth middleware | mitigate | JWT signature verified via verifyToken in auth.ts; Bearer token required; 401 on missing/invalid |
| T-02-02 | Elevation of Privilege | PATCH /words/:id | mitigate | getWordForUser checks word.userId === req.user.userId before update; returns 403 on mismatch |
| T-02-03 | Elevation of Privilege | DELETE /words/:id | mitigate | Same ownership check via getWordForUser before delete |
| T-02-04 | Tampering | POST /words body | mitigate | headword validated as non-empty string; userId taken from req.user (JWT), never from request body |
| T-02-05 | Information Disclosure | GET /words | mitigate | prisma.word.findMany filtered by userId — user sees only their own words |
| T-02-06 | Denial of Service | POST /words rapid fire | accept | Global rate limiter (200 req/15min) in index.ts applies; word creation is low-frequency user action |
</threat_model>
