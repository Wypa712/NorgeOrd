## Spacing Scale

Declared values (multiples of 4 only). Map to Tailwind utility classes:

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, badge internal padding |
| sm | 8px | `p-2` / `gap-2` | Compact row internal spacing |
| md | 16px | `p-4` / `gap-4` | Default element spacing, form field gaps |
| lg | 24px | `p-6` / `gap-6` | Drawer/modal inner padding |
| xl | 32px | `p-8` / `gap-8` | Section breaks |
| 2xl | 48px | `py-12` | Major section breaks |
| 3xl | 64px | `py-16` | Page-level empty state centering |

**Exceptions:**
- Touch targets (FAB, row tap area, drawer close): minimum 44px height — use `min-h-[44px]` or DaisyUI `btn` which meets this by default
- List row: `py-3` (12px vertical) is acceptable — compact layout per D-01
- AppShell main: `px-4 py-6` (established in Phase 1 — do not change)

## Typography

DaisyUI nord theme inherits system-ui font stack. No custom font loaded.

| Role | Size | Tailwind | Weight | Tailwind | Line Height | Usage |
|------|------|----------|--------|----------|-------------|-------|
| Body | 16px | `text-base` | 400 regular | `font-normal` | 1.5 | List row text, drawer body, form labels |
| Label / Small | 14px | `text-sm` | 600 semibold | `font-semibold` | 1.4 | Badge text, form field labels (`label-text font-semibold` — Phase 1 pattern), helper text |
| Heading | 20px | `text-xl` | 600 semibold | `font-semibold` | 1.2 | Drawer title (headword), section headings |
| Metadata | 12px | `text-xs` | 400 regular | `font-normal` | 1.3 | Timestamps, secondary info in detail view |

**Rules:**
- Maximum 4 sizes in use — enforce this. Do not introduce `text-2xl` or larger in Phase 2.
- Maximum 2 weights: 400 and 600. Do not use 500 or 700.
- Headword in list row: `text-base font-semibold` (not heading size — compact layout).
- Translation in list row: `text-sm text-base-content/70` (muted, secondary).

## Interaction Contracts

### FAB "+" button
- Position: `fixed bottom-6 right-6 z-50`
- Size: `btn btn-circle btn-primary w-14 h-14` (56px, meets 44px touch target)
- Label: `+` (text-2xl) — no text label, aria-label="Додати слово"
- Action: sets `addDrawerOpen = true` → renders AddWordDrawer with `modal-open`

### Word row tap
- Full row is clickable — wrap in `<button>` or set `onClick` on `<li>`
- Tap → sets `selectedWordId` → opens WordDetailDrawer in view mode
- No navigation — URL stays at `/words` (D-11, D-23)

### Drawer open/close
- State: local `useState` in WordsPage — `addDrawerOpen: boolean`, `selectedWordId: string | null`
- Open: toggle boolean / set id
- Close: click backdrop (modal DaisyUI default) OR explicit "Скасувати" button OR after successful save/delete
- No URL change (D-11)

### Edit mode toggle
- Default: view mode in WordDetailDrawer
- "Редагувати" button (`btn btn-ghost btn-sm`) in modal-action → sets `editMode = true`
- Switches same modal content to editable form
- "Скасувати" in edit mode → `editMode = false` (back to view, no save)

### Save (Add / Edit)
- Submit → `useMutation` → optimistic or await → invalidate `['words']` query
- Success: close drawer + `toast.success('Слово збережено')`
- Error 409 (duplicate headword): `toast.error('Слово «{headword}» вже існує у вашому словнику')`
- Error other: `toast.error('Не вдалося зберегти слово. Спробуйте ще раз.')`

### Delete
- "Видалити" button in view mode → sets drawer state to `confirm-delete`
- Confirm "Видалити" → `useMutation DELETE` → on success: close drawer + invalidate query + `toast.success('Слово видалено')`
- "Скасувати" in confirm → back to view mode (no close)

### Loading states
- Word list loading: show DaisyUI skeleton rows (`animate-pulse bg-base-300 rounded h-12`) × 5
- FAB during pending add: disable FAB (`disabled` attribute)
- Save button: `loading` prop on Button component (Phase 1 pattern)
- Delete button: `loading` prop on Button error variant

## Accessibility Contract

- All form fields: `id` + `htmlFor` pairing (established in Input.tsx — follow same pattern for SelectField)
- Modal: use `<dialog>` element or DaisyUI `modal` with `role="dialog"` and `aria-modal="true"`
- FAB: `aria-label="Додати слово"` (no visible text label)
- Delete confirm: focus trap within modal — DaisyUI modal handles this by default
- Word row as button: `role="button"` or wrap in `<button>` — must be keyboard-activatable
- Error messages: `role="alert"` on toast (established in Toaster component from Phase 1)
- Loading states: `aria-busy="true"` on button (established in Button.tsx)
- Badge colors: do not rely on color alone — badge always includes text label
